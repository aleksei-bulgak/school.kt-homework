# School.kt

## Menthor task
### Description
Это  будет проект из двух приложений
Первое приложение  будет  представлять  из  себя  сервер  который  будет  собирать  и  накапливать  информацию  из  внешнего  источника.  Накапливать  инфу  можно  как  в  просто память  так  и  в  базу  данных.  Я  бы  рекомендовал  базу  прикрутить,  что  бы  данных  можно  было  по  больше  собрать.

Этот  ваш  сервер  так  же  будет  иметь  некий  REST интерфейс  по  которому  к  нему  будет  подключаться  клиент.  В  REST  интерфейсе  сервера  будет  не менее 3х эндпоинтов:
-    Какая-то  агрегация  суммарная  по  всем  данным  что  есть  на  сервере
-    Запрос  определенной  агрегации  по  конкретной  сущности
-    Запрос  данных  по  какому-то  ключу

У  севрера  должна быть дока по API  которая  автоматически  генерится.
Тут  смотрите  в  OpenAPI  тк.  схему  придется  писать  руками  а  потом  у  же  по  ней  генератор  запускать
Касательно  клиента.  Я  предлагаю  что  бы  это  было  консольное приложение.
Вы  будете  запускать  его  из  консоли  с  аргументами  и  оно  будет  печтать  ответ.
Обязательно  нужно  использовать  правильные  тулзы  для  парсинга  аргументов  и  уметь  показывать  справку  по  -h
Вывод  в  консоль  нужно  делать  в  цвете.  И  красиво  данные  в  табличках  показывать.
Если  есть  желание,  то  можно  клиента  сделать  в  виде  Android  приложения  или  KotlinJS  веб  приложения,  но  тут  с  имплементацией  я  не  особо  смогу  помочь.
Как  вы  видите  требования  довольно  размытые,  потому,  что  данные  которые  вы  будете  обрабатывать  разные  и  тут  я  хочу,  что  бы  вы  выбрали  сами,  что  хотите.

Я  предлагаю  использовать такие API:
-    cryptocompare.com
-    github.com
-    twitter.com
-    instagram.com  (возможно)

Можете  предложить  и  свой  вариант,  самое  главное  что  бы  там  было  много  связанных  данных,  которые  легко  взять  без  дополнительных  пермиссий.

После  того  как  вы  определитесь  с  источником  данных  -  буду  с  вами  лично  уже  думать  над  тем  что  доставать  накапивать  на  вашем  сервере.
Сейчас  же  основная  задача  для  вас  изучить  API источника данных и попробовать  сделать  прототип  сервера  и  клиента.  Чисто  два  приложения  заглушки,  но  которые  уже  могут  друг  с  другом  общаться.  Клиент,  например  может  по  ключу  -v  показать  свою  версию,  а  по  ключу  --server-version  показать  ответ  с  информацией  версии  сервера  и  что-нибудь  еще
Естественно проекты это grale + kotlin dsl
И нужно пытаться использовать как можно меньше Java библиотек при наличии альтернатив на kotlin или хотя бы той же wrapper под котлин (edited) 
Не забудьте везде использовать логирование (есть либа обертка над slf4j для котлина) (edited)
